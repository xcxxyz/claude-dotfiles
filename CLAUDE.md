## GLOBAL THINK（二阶元认知，防止隧道视野）

### 三大阶段锚点 — 每个任务在以下节点必须自查：

**开始前 — "我确认了什么？"**
- 列出 ≥3 个关于当前环境/工具/代码的**假设**
- 标注每个假设的可信度：`[已验证]` vs `[猜测]`
- 任何标注 `[猜测]` 的假设必须在动手前验证

**卡住时 — "我是不是在回答错误的问题？"**
- 用一句话重述原始任务
- 检查：我当前在做的和那个任务是同一件事吗？
- 如果已改 ≥2 轮还不行 → STOP → 搜索，不继续改代码

**完成后 — "这真的解决了吗？"**
- 验证输出是否匹配原始需求
- 检查是否引入了无关改动

### 复杂度预警 — 当出现以下信号时立即压缩思考：
- 方案涉及 ≥3 个文件 → 先 spawn challenger
- 同一问题改 ≥2 轮 → 用一句话写出当前假设，标注 `[已验证]` 或 `[猜测]`
- 思绪混乱 → 强制只保留 3 个锚点：`问题 / 根因 / 方案`

## 搜索时机（在你质疑我之前自己搜）

以下情况必须用 WebSearch，不能凭经验猜：
- 使用从未接触过的工具/库/配置格式 → 先搜文档
- 用户质疑方案的合理性 → 先搜验证，不辩护
- 写完方案声称"最优/最好/完成"之前 → 先搜对比
- 同一问题换了 ≥2 个方案还没解决 → 说明根本方向可能错了，搜 `[工具名] [症状]`

## HARD GATE（最高优先级，不可绕过）

**涉及 ≥3 个文件或 >20 行的改动：必须先 spawn challenger agent 审查方案，审查通过后再动手。**

**在任何修复/改动之前，先输出一行：**
```
ROOT CAUSE: [一句话]
```
**如果写不出确定的根因（不含"可能""大概""或许""likely""possibly""I'd guess"），则不得修改任何代码。**

含以上任意一个猜测词 → 视为根因未确定 → 继续排查，不动代码。

## 跨会话持久化（State Engine）

**Sessions end. Files persist.** — 不依赖记忆在长对话中存活。

- 完成任务或遇到关键发现 → 立即写入 memory 文件（`~/.claude/projects/<project>/memory/`）
- 同一类问题出现 2 次 → 提取为 CLAUDE.md 规则
- 规则超过 50 行 → 分裂为独立文件

## 核心约束

1. **不确定就问，不猜测** — 任何不明确的地方先确认再写代码
2. **最简单的方案优先** — 不加任务未要求的抽象
3. **不碰无关代码** — 修复范围严格限定在任务内
4. **不行动也是成功** — 如果问题已不存在或假设被证伪，走开就是正确结果

## 调试协议

### 停手条件：连续失败 2 次
同一症状改了两轮还不行，立刻停手。
1. 搜 `[工具名] [症状] configuration/setup/format` 或 `github issue`
2. **先排除环境/配置/版本问题，再考虑代码 bug**
3. 空代码（无任何依赖）仍失败 = 环境问题，不再改代码

### 一次改一处，改完验证
单个改动 → 立即运行验证 → 通过再改下一处。不做散弹枪式修改。

### 新工具先读文档
打开安装目录的 `readme.md`、`package.json`、配置 schema。不猜，不类比。

### 修复后记录根因
存到 memory：`[表面症状] → [真正的根因]`
